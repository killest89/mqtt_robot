当前发送指令(01050002FF002DFA)会触发自动模式，巡线+监测黄线系列动作+监测绿线逻辑，现在需要在原来基础上增加一个手动模式功能，具体是
1. 服务器侧发送的十六进制指令(01050002FF002ABA)则开启手动模式开始巡着红线走，等发停止指令(01050002FF002BCB)则停车；2. 手动模式过程碰到黄色不用暂停，后续需要支持发各种继电器指令手动控制升降和云台转向（自动模式监测到黄线会固定执行系列继电器动作，手动模式不需要，而通过发送指定命令来触发）；
3. 发指令(002FF002CCC)可采集温度数据报送(不用循环采集，采一次即可)
4. 手动模式和自动模式在监测到终点绿色点逻辑保持一致，卸掉进程自动去回充。
5. 当发送目前支持的指令(01050002FF002DFA)才去自动模式，保持现有逻辑，即走到黄色点自动执行系列动作和采集温度动作，绿线回充等
6. 后续有自动模式的开车和停车，以及手动模式的开车停车几种模式，然后自动模式的停车保留现状暂停+关灯，手动模式的停车只需要暂停运行就行，不需要关灯，具体如下：
指令	模式	停车行为
01050002FF002DFA	自动开始	开灯 + 巡线（黄线停车+继电器，绿线回充）
01050002000028CA	自动停止	停车 + 关灯
01050002FF002ABA	手动开始	开灯 + 巡线（忽略黄线，绿线回充）
01050002FF002BCB	手动停止	停车 + 保持灯光
